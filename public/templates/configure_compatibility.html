    Other than explicitly written actions (which are functional requirements), the system should also provide certain interface or functionality for user
    For example, certain system will provide RESTful API which is not an action that user could perform, however, it is needed for the system to perform its normal functionalities
    These functional constraint will describe how actions are performed by the system

    For example, the most commonly introduced functional constraints are:
    - System API or console commands
    - Linked action (which one action triggers another)
    - Dependencies (such as dependency on other framework)
